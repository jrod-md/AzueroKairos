## Qué NO debe subirse como entrega oficial

Estos documentos no deben subirse como entrega final de TAIKAI ni presentarse como resultado oficial construido. Tampoco deben sustituir el README oficial, pitch final, dashboard, código, Evalscripts finales, resultados procesados oficiales ni documentación final de competencia.

El repositorio oficial, el código oficial, la UI oficial, los outputs oficiales y los documentos finales deben crearse durante la ventana oficial del hackathon.

## Qué queda pendiente antes de mostrarlos a profesora

1. Revisar nombres de autores/equipo y fecha.
2. Confirmar si la profesora prefiere enfoque académico, competencia o ambos.
3. Validar si se requiere citar fuentes locales sobre Río La Villa, inocuidad agrícola o agroexportación.
4. Revisar que el lenguaje de limitaciones sea suficientemente claro para no parecer detector químico.
5. Decidir si se convertirán a DOCX/PDF con formato institucional UTP.
6. Revisar que cualquier mención a resultados técnicos se entienda como spike descartable, no entrega final.
